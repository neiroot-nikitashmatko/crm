# LOCAL RUN

Пошаговый запуск локально. Команды из **корня проекта** (`PROclients/`), если не указано иное.

Браузер после запуска: **http://localhost:5173**

---

## Первый запуск (с нуля)

### 1) PostgreSQL

Если контейнера ещё нет:

```bash
docker run --name proclients-postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=proclients \
  -p 5432:5432 -d postgres:16
```

Если контейнер уже создавали раньше:

```bash
docker start proclients-postgres
```

### 2) Миграции

```bash
chmod +x backend/scripts/apply_migrations.sh
./backend/scripts/apply_migrations.sh
```

Скрипт по порядку выполняет `backend/migrations/*.up.sql` (001–017). Альтернатива — вручную через `psql` или TablePlus.

**Важно:** скрипт рассчитан на **пустую** БД. Повторный запуск на той же базе упадёт — не запускайте его снова без необходимости.

### 3) Первый администратор (один раз)

После миграций в БД ещё нет пользователей для входа. Создай админа SQL-запросом — **пароль только здесь, не в git**.

Шаблон: `backend/scripts/create_admin.sql.example` (скопируй, подставь телефон, пароль, должность и дату рождения, выполни в psql или TablePlus).

Локально (Docker):

```bash
docker exec -i proclients-postgres psql -U postgres -d proclients \
  < путь/к/твоей_копии_admin.sql
```

Перед этим отредактируй **копию** файла с **своими** данными или вставь SQL прямо в TablePlus.

Телефон — формат `+79001234567`. Роль `admin` — полный доступ.

### 4) Backend (терминал 1)

```bash
cd backend
cp -n .env.example .env   # только если .env ещё нет
go mod tidy
go run ./cmd/server
```

В `backend/.env` обязательны `DATABASE_URL` и `JWT_SECRET` (минимум 32 символа). Пример — в `backend/.env.example`.

API: **http://localhost:8080**

### 5) Frontend (терминал 2)

```bash
cd frontend
cp -n .env.example .env   # только если .env ещё нет
npm install               # первый раз или после смены зависимостей
npm run dev
```

`frontend/.env`: `VITE_API_BASE_URL=http://localhost:8080` (уже в `.env.example`).

### 6) Вход в приложение

Открой **http://localhost:5173** и войди под созданным администратором.

## Повторный запуск (уже настраивали проект)

```bash
docker start proclients-postgres
```

Терминал 1 — backend:

```bash
cd backend
go run ./cmd/server
```

Терминал 2 — frontend:

```bash
cd frontend
npm run dev
```

Миграции и создание админа — **не нужны**, если база уже настроена.

**Новые миграции** (если в репозитории появился файл `018_...`, `019_...` и т.д.) — примени только новый `.up.sql` вручную, не перезапускай весь `apply_migrations.sh` на существующей базе:

```bash
docker exec -i proclients-postgres psql -U postgres -d proclients \
  < backend/migrations/018_allow_lead_attachments_and_activities.up.sql
```

После изменений в **backend** — перезапусти `go run ./cmd/server` (Ctrl+C и снова запуск).

## Сотрудники (через UI)

Раздел **Сотрудники** доступен только администратору:

- **Список сотрудников** — данные из PostgreSQL (`GET /api/v1/users`)
- **Добавить сотрудника** — создание в БД (`POST /api/v1/users`)
- **Редактирование / удаление** — `PATCH` и soft-delete (`DELETE`) через API

Моковые записи не используются: все операции идут через backend → PostgreSQL.

---

## После изменений в коде

- **Backend** или **миграции** — перезапусти `go run ./cmd/server`
- **Frontend** — обнови страницу (лучше Cmd+Shift+R)
- **Новая миграция** — примени только новый `.up.sql` (не перезапускай весь скрипт на старой БД без понимания последствий)
